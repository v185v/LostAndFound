# 前后端分离架构
## 不是最终版本，随时修改


后端仓库地址:https://github.com/v185v/LostAndFound

# 校园失物招领系统

在本题目中，有学生/老师、失物招领管理员、系统管理员三个模块。

**一、学生/老师**

1、登录：使用学号/工号及统一密码（初始为身份证后六位）登录。首次登录需强制修改密码。登录后先显示失物招领相关通知与公告，确认后进入主界面。

2、查询信息：可按物品类型、丢失/拾取地点、时间范围、物品状态进行筛选查询。点击查看物品详情，包括物品描述、特征、拾取/丢失时间、存放地点、联系方式（仅对管理员和失主/拾主可见）。

3、发布信息：发布失物信息时，需要填写物品名称、类型、丢失地点、丢失时间、物品特征、联系人及联系电话、是否有悬赏（可选），可上传物品相关照片（如有）。发布招领信息时，需填写物品名称、类型、拾取地点、拾取时间、物品特征、联系人及联系电话，必须上传物品清晰照片，便于失主确认。

4、管理发布记录：在“我的发布”中，查看所有发布的失物或招领记录，状态包括：“待审核”“已通过”“已匹配”“已认领”“已驳回（显示原因）”“已取消”。对于“待审核”状态的记录，可修改或删除；对于“已通过”但未完成认领的记录，可补充信息或取消发布。

5、认领与沟通：若发现招领信息中物品为自己丢失的，可通过系统提交认领申请，说明物品额外特征或相关证明信息。可查看认领申请进度，与管理员或拾主进行有限沟通。

6、退出登录。

**二、失物招领管理员**

1、登录：使用分配的工号及密码登录。登录后先显示系统通知与公告，确认后进入主界面。

2、审核发布信息：查看待审核失物/招领信息列表，审核内容包括信息真实性、完整性、照片清晰度等，做出“通过”或“驳回”决定，驳回必须填写理由。可查看历史审核记录。

3、管理物品状态：维护已发布信息的状态，当物品完成认领后，及时将状态更新为“已认领”；对长期无人认领的物品（如超过30天），标记为“已归档”，并记录物品处理方式。可将虚假、重复的信息标记为无效并删除。

4、信息维护与查询：更新物品存放地点、联系方式等信息；总览失物招领信息的发布、匹配、认领统计情况；支持按多种条件查询历史记录，导出统计数据（可选）。

5、退出登录。

**三、****系统管理员**

1、登录：使用超级管理员账号登录。

2、 全局管理权限：查看全校失物招领信息总览、统计数据。修改系统参数（如物品类型分类、认领时效等）；设置信息发布的权限限制（如发布频率、内容规范）。

3、账号与权限管理：查看所有学生/老师和失物招领管理员的账号信息；新增、禁用失物招领管理员账号；向指定用户或全体用户发送系统通知。

4、公告与内容管理：发布全局公告（如系统维护通知等）；审核管理员发布的区域公告；删除违规、虚假的发布信息或评论（若系统有评论功能）。

5、数据管理：备份、导出全校失物招领系统数据；清理过期无效数据；处理用户投诉与反馈（如信息错误、恶意发布等）。

6、退出登录。
